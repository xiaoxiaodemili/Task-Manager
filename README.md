# Task-Manager
Task Manager for team &lt;100
# 任务管理系统：业务逻辑与技术实现全景梳理

写在前面：当初做这个工具的想法其实挺简单。市面上流行的任务协同软件要么太重、要么依赖外网，对于部分需要在局域网内快速协同的小团队来说，反而成了一种负担。所以我们做了一个“开箱即用、轻量级、颜值上线”的任务看板系统。

这篇文章主要就是把这套系统“能干嘛”以及“底子是怎么搭的”给彻底掰扯清楚。

---

## 1. 系统到底能解决什么问题？（核心业务逻辑）

说白了，团队协作最大的麻烦就是“责任不清、进度不明”。这个系统主要通过四个核心模块把这件事理顺：

### 1.1 权限与角色（谁能干什么）

系统里一共就三种角色，分工非常明确：

- **超级管理员 (Admin)**：拥有上帝视角。比如有个新项目立项了，超管可以直接建项目，指派具体由哪位项目经理（PM）来负责。团队的人员名单、角色设定也都归超管管。
- **项目经理 (PM)**：主要负责管理具体的业务。PM 可以创建项目、设置项目截止日期、邀请项目成员。在项目内，PM 可以自由地建任务、分派任务给组员，并监控整体进度。
- **普通成员 (MEMBER)**：主要属于“干活”的一方。登录后可以看到自己参与的项目（无论是作为负责人、成员还是有具体任务分配）。处理完手头的任务后，将卡片拖至“已完成”即可增加项目完成度。

### 1.2 搞定日常工作（项目与看板流转）

- **项目隔离**：每个项目都是一个独立的池子。大家点进具体的项目，就能看到一个熟悉的 Kanban（看板）。
- **拖拽体验**：把任务分成了 **待处理 (TODO)**、**进行中 (IN_PROGRESS)** 和 **已完成 (DONE)**。大家每天的工作就是在看板上把卡片移来移去。
- **死线警告 (Deadline)**：没人喜欢记日子，所以系统会自动帮大伙盯着。项目和任务均支持设置截止日期。离截止日期还剩一两天时，卡片上会自动弹出橙色的警告；一旦逾期，就直接爆红，哪怕不点开也能一眼看清。
- **项目成员制**：项目不再是孤岛。创建项目时可以勾选多名参与成员，确保团队信息同步，成员无需等待任务分配即可提前关注项目动态。
- **可视报表中心**：针对管理层设计的明细化报表。支持按月、按季、按年实时筛选查看所有任务的执行明细（包含任务描述、状态、责任人及截止/完成时间），并支持一键导出符合标准的 CSV 文件供离线汇总。

## 2. 这个系统是怎么写出来的？（技术架构）

为了达到“极简部署”的目的，系统在技术底层完全抛弃了那些笨重的东西。

### 2.1 后端：Node.js 加个小巧的 SQLite

- **只用一个文件当数据库**：没有搭建 MySQL 或 Redis 服务器的折腾。所有的数据全都存在同级目录下的一个 `taskmanage.db` 文件里。连表结构（用户、项目、任务卡片及成员关联表）在系统第一次启动时都会自动建好，并支持平滑的版本升级迁移。
- **API 与鉴权**：后端主要依靠 Express 框架提供数据接口，同时使用了 JWT（JSON Web Token）。只要一登录，后端就发个无状态的门禁卡（Token）。
- **密码安全**：虽然是局域网自用，但用户的密码我们依然用了 bcrypt 做了单向加密（哈希处理），数据库哪怕被人拿走也看不到明文密码。
- **数据联查优化**：后端针对复杂的报表统计需求，采用了高效的 SQL 联表查询逻辑，确保即使在数据量增加时也能快速响应前端的筛选请求。

### 2.2 前端：不堆砌框架的 Vanilla UI

- 这个项目的前端完全是原生的 HTML、CSS 另加一个 `app.js` 搞定的。没上 Vue 也没用 React。
- **为什么页面看起来这么现代？** 主要是因为用了一套叫做 Glassmorphism（毛玻璃）的视觉设计风格。利用 CSS 的半透明高斯模糊配合极高对比度的字体和色块，在视觉上显得非常有层次和通透感。
- **流动的页面交互**：虽然不是严格的复杂框架，但我们通过 JS 控制 DOM 的显示和隐藏，做出了类似 SPA（单页面应用）的顺滑体验。所有的新建、编辑弹窗也都是动态拼接生成的，轻巧好打理。

### 2.3 贴心的部署设计：跟“常驻黑窗口”说再见

以往咱们用 Node.js 跑个服务，桌面总得留个黑色的命令行窗口，不小心点个叉服务就挂了。
为了解决这个历史遗留问题，我们顺手写了两个 `.bat` 的脚本组合：

- **怎么做到的**：底层其实是用 Powershell 的隐藏窗口模式启动的 Node 进程，并且顺滑地把运行时产生的进程号码（PID）记在了一个小文件里 (`server.pid`)。
- 当你需要关掉服务时，程序一扫那个 PID 就能精准狙击退流，完全不会误伤电脑里正在跑的其他 Node 服务。从技术上彻底做到了安全、静默地后台驻留。
